Q1. Difference between map, filter, forEach and find?

Ans: `map`: The map() method creates a new array with the results of calling a provided function on every element in the calling array.
The original array is not modified. The function passed to map() should return a new value, which will be added to the new array.

`forEach : ` .forEach(), is used to execute the same code on every element in an array but does not change the array and it returns undefined.
The function does not return a value. It simply performs some operation on each element of the array.

`filter: ` The filter() method creates a new array with all elements that pass the test implemented by the provided function.
The original array is not modified. The function passed to filter() should return a boolean value,
indicating whether or not the element should be included in the new array.

`find: `The find() method returns the value of the first element in the array that satisfies the provided testing function. Otherwise,
undefined is returned. The original array is not modified. The function passed to find() should return a boolean value,
indicating whether or not the element satisfies the search criteria.

Q2.Difference between local storage session storage and cookies?

Ans:Local storage is useful for storing data that the user will need to access later, such as offline data.
Session storage is a great way to improve the performance of your web applications.
Cookies are a good choice for storing data that should not be persisted for a long time, such as session IDs.

Difference:

Data storage: Cookies, local storage, and session storage all have different limits on the amount of data they can store. Cookies are limited to 4KB of data, while local storage and session storage can store up to 10MB of data.
Data expiration: Cookies can have an expiration date set, while local storage and session storage do not. This means that cookies can persist even after the browser is closed, while local storage and session storage will be cleared when the browser is closed.
Data accessibility: Cookies can be accessed by both the server and the client-side scripts, while local storage and session storage can only be accessed by the client-side scripts.
Data transmission: Cookies are automatically sent with every HTTP request, while local storage and session storage are not. This means that cookies can be used to send data to the server with every request, while local storage and session storage are only accessible on the client-side.

Q3.How the event loop works in JavaScript?

Ans : JavaScript maintains two main data structures: the Call Stack and the Event Queue.
When the JavaScript engine executes a function, it adds it to the Call Stack.
If the function returns a value, it is popped off the Call Stack.
If the function initiates an asynchronous operation, the operation is offloaded to a separate thread, and the function is popped off the Call Stack.
When the asynchronous operation is complete, a corresponding event is added to the Event Queue.
The Event Loop continuously checks the Event Queue and adds any events to the Call Stack.
If the Call Stack is empty, the Event Loop pops the next event off the Event Queue and adds it to the Call Stack.
The corresponding function associated with the event is then executed.

Q4.What quiz timer function do ?

Ans: A quiz timer is a feature that can be implemented to limit the amount of time a user has to complete a quiz.
In Js Code quizTimer takes one parameter called dismiss, which is a boolean value that determines whether to stop or start the timer.
When the dismiss parameter is true, the function stops the timer by calling clearInterval on the timer variable and then returns from the function.
When the dismiss parameter is false, the function starts the timer by calling setInterval and passing in a callback function that will run every second. Inside the callback function, the function updates the time display by calculating the minutes and seconds from the count variable, which increments every second. The function also checks whether the count variable has reached 60 seconds and alternates the color of the timer text every 2 seconds.

Q5.What choose quiz function do ?

Ans : If a website offers quizzes on different topics, the "choose quiz" function would allow the user to select which topic they want to be quizzed on.

The "choose quiz" function first checks if an object with the same id as the current quiz exists in an array called answers.
If it does, the function updates the object's givenAns property with the new answer provided.
If it doesn't exist, the function creates a new object with the quiz information and answer provided and pushes it to the answers array.
After updating or pushing the new answer to the answers array, the function calls another function called displayAnswers and passes the updated answers array to it.
